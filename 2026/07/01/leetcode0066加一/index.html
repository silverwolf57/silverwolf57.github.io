66. 加一

题目描述

给定一个表示大整数的整数数组 digits，其中 digits[i] 是整数的第 i 位数字。这些数字按从左到右，从最高位到最低位排列。这个大整数不包含任何前导 0。

将大整数加 1，并返回结果的数字数组。

示例

  • 示例 1：
    • 输入： digits = [1,2,3]
    • 输出： [1,2,4]
    • 解释： 输入数组表示数字 123。加 1 后得到 $123 + 1 = 124$。
  • 示例 2：
    • 输入： digits = [4,3,2,1]
    • 输出： [4,3,2,2]
    • 解释： 输入数组表示数字 4321。加 1 后得到 $4321 + 1 = 4322$。
  • 示例 3：
    • 输入： digits = [9]
    • 输出： [1,0]
    • 解释： 输入数组表示数字 9。加 1 得到了 $9 + 1 = 10$。

提示

  • $1 \le \text{digits.length} \le 100$
  • $0 \le \text{digits}[i] \le 9$
  • digits 不包含任何前导 0

思路一：模拟高精度加法

核心思想

模拟手工竖式进位加法，从最低位开始 +1，逐位计算本位、更新进位，结果逆序存入临时数组，遍历结束若还有进位则补充，最后反转数组得到答案。

复杂度分析

设数组长度为 $n$。

  • 时间复杂度： $O(n)$
    遍历一遍数组 $O(n)$，反转数组也是 $O(n)$，总线性复杂度。
  • 空间复杂度： $O(n)$
    额外开辟 res 数组存储结果，最坏长度 $n+1$；若不算返回数组空间，额外辅助变量仅常数 $O(1)$。

思路二：找第一个非 9 数字优化（最优常规解法）

核心思想

逆序遍历：

  1. 当前位不等于 9： 本位 +1，直接返回原数组；
  2. 当前位等于 9： 本位置 0，继续向前进位；
  3. 遍历完所有位全为 9： 数组扩容一位，首位赋值为 1。

复杂度分析

设数组长度为 $n$。

  • 时间复杂度： $O(n)$
    最坏情况（全部是 9）需要完整遍历整个数组；平均情况大概率中途提前返回，实际效率最高。
  • 空间复杂度： $O(1)$（最优）
    直接原地修改原数组；只有全 9 特例才新建数组，不计返回存储空间时，额外空间为常数级别。

思路三：Python 类型转换极简写法

核心思想

利用 Python 支持超大整数的特性，完成转换链：
数字列表 $\rightarrow$ 字符串拼接 $\rightarrow$ 整数 +1 $\rightarrow$ 转字符串 $\rightarrow$ 转回数字列表。

复杂度分析

设数组长度为 $n$。

  • 时间复杂度： $O(n)$
    多次遍历转换：列表转字符串、字符串转整数、数字再转回字符串、再转回列表，每一步都是线性 $O(n)$。
  • 空间复杂度： $O(n)$
    需要额外字符串、中间整数、结果列表占用线性空间；

局限性： 题目数组最长可达 100 位，绝大多数编程语言（C++/Java）原生整型存不下，该写法仅 Python 可用，通用性差。
